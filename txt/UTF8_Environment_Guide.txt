# 🛠️ [긴급/필독] 윈도우 API 한글 깨짐 및 UTF-8 환경 통합 가이드

팀원 여러분, 반갑습니다! 현재 라이더(Rider)와 비주얼 스튜디오(VS) 간의 인코딩 불일치로 인해 일부 환경에서 윈도우 API 오류나 한글 깨짐 현상이 발생하고 있습니다. 프로젝트의 원활한 빌드를 위해 **모든 팀원은 반드시 아래 3가지 설정을 똑같이 맞춰주시기 바랍니다.**

---

## 1. 프로젝트 문자 집합 설정 (필수)
비주얼 스튜디오가 문자열을 다루는 기본 방식을 '유니코드'로 통일해야 합니다.

1. 솔루션 탐색기에서 **[P_Harem]** 프로젝트 우클릭 -> **[속성 (Properties)]**
2. 좌측 메뉴: **[구성 속성 (Configuration Properties)]** -> **[고급 (Advanced)]**
3. 항목 중 **[문자 집합 (Character Set)]**을 찾습니다.
4. 값을 **[유니코드 문자 집합 사용 (Use Unicode Character Set)]**으로 변경합니다.
   - ⚠️ '멀티바이트 문자 집합'으로 되어 있으면 한글 처리가 되지 않습니다!

---

## 2. 컴파일러 UTF-8 강제 옵션 추가 (강력 권장)
Rider에서 작성된 UTF-8 소스 코드를 VS 컴파일러가 오해하지 않도록 명시해 줍니다.

1. 프로젝트 **[속성]** 창을 계속 엽니다.
2. 좌측 메뉴: **[C/C++]** -> **[명령줄 (Command Line)]**
3. 하단의 **[추가 옵션 (Additional Options)]** 칸에 아래 내용을 직접 타이핑하여 입력합니다.
   - `/utf-8`
4. [확인]을 눌러 저장합니다.

---

## 3. 소스 파일 저장 인코딩 확인
만약 특정 파일 내부의 한글 주석이 깨져 보인다면, 해당 파일의 저장 형식을 바꿔야 합니다.

- **VS 2022 사용자**: [파일] -> [다른 이름으로 저장] -> [저장] 버튼 옆의 화살표 클릭 -> [인코딩하여 저장] -> **[Unicode (UTF-8 with signature) - Codepage 65001]** 선택 후 저장.

---

## 💡 왜 이렇게 해야 하나요? (매니저의 조언)
- 우리 게임은 외부 대본 파일(`Scenario.txt`)을 **UTF-8**로 읽어옵니다.
- 콘솔 역시 실행 시 `chcp 65001` 명령을 통해 **UTF-8** 모드로 작동합니다.
- 따라서 소스 코드와 프로젝트 설정이 모두 **UTF-8(유니코드)**로 일치되어야만 윈도우 API가 데이터를 정확하게 전달받아 렌더링할 수 있습니다.

설정 중 어려움이 있다면 언제든 수석 프로그래머(사용자)에게 문의해 주세요!
