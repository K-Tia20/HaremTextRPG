# ✨ 팀 모태쏠로 시나리오 작성 가이드 (V2.0)

이 문서는 고해상도 콘솔 엔진의 **로그창(LOG)** 레이아웃을 보호하고, 시나리오 작가님이 가장 효율적으로 대사를 작성할 수 있도록 돕는 지침서입니다.

---

## 1. 기본 양식 (Strict Format)

모든 데이터는 반드시 아래의 형식을 지켜야 합니다.

> **`KEY : "대사 내용"`**

*   **KEY**: 프로그램 코드에서 부르는 이름입니다. (영문 대문자, 숫자, 언더바 `_` 사용 권장)
*   **콜론(`:`)**: 키와 내용 사이에는 반드시 콜론이 있어야 합니다.
*   **따옴표(`" "`)**: 내용은 반드시 쌍따옴표로 감싸야 합니다.
*   **줄바꿈(`\n`)**: 내용 안에서 줄을 바꾸고 싶다면 반드시 `\n` 문자를 사용하세요.

---

## 2. 로그창 보호 규칙 (Safety Rules)

현재 엔진의 로그창 너비는 **약 116칸(한글 기준 약 58자)**입니다. 이 범위를 넘어가면 UI 테두리가 부서질 수 있습니다.

### ✅ 권장 작성 요령
*   **한 줄 길이 제한**: 한 줄에 한글 기준 **50자 이내**로 작성하고, 그보다 길어지면 `\n`을 넣어 줄을 바꿔주세요.
*   **입력 태그 활용**: 
    *   `[PLAYER_NAME]`: 플레이어 이름으로 자동 치환됩니다.
    *   `[INPUT]`: 해당 위치에서 사용자의 입력을 기다립니다. (커서 자동 배치)

### ❌ 주의 사항 (하지 마세요!)
*   **긴 문장 무지성 나열**: `\n` 없이 너무 긴 문장을 쓰면 로그창 우측 벽을 뚫고 나갑니다.
*   **특수 문자 과용**: `╔, ═, ║` 같은 아스키 아트를 쓸 때는 너비 계산에 주의하세요. (총 110칸 이내 권장)

---

## 3. 작성 예시

```txt
# [좋은 예]
INTRO_SAY : "안녕?\n만나서 반가워. 내 이름은 [PLAYER_NAME]이야."

# [나쁜 예 - 너무 길어서 벽을 뚫음]
BAD_EXAMPLE : "이 문장은 너무나도 길어서 로그창의 테두리를 부수고 나갈 것이며 프로그램의 UI를 엉망으로 만들고 말 것입니다 제발 줄바꿈을 해주세요."
```

---

## 4. 엔진 업데이트 사항 (작가님 필독)

이제 엔진이 **자동 줄바꿈(Auto-Wrap)**을 지원하지만, 가독성을 위해 작가님이 직접 `\n`을 넣어 리듬을 조절해주시는 것이 가장 아름답게 출력됩니다. 

또한, `\u001b[31m`과 같은 색상 코드를 자유롭게 섞어 쓰셔도 무방합니다. (색상 코드는 글자 수 계산에서 자동으로 제외됩니다.)
